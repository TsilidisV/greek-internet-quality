## 📐 Architecture

At its core, this project implements an event-driven, serverless **Medallion Data Lakehouse** on Google Cloud Platform, fully provisioned via **Terraform**. The pipeline is orchestrated by **GitHub Actions**, which first triggers a Python ingestor script which transforms JSON payloads into parquet files (enforcing a schema-on-write approach) and saving them to a **Google Cloud Storage** bucket (the Bronze layer). Upon successful ingestion, a containerized **PySpark** job spins up on demand via **Google Cloud Run** (triggered by Github Actions) to process the data into **BigQuery**. Here, the data is progressively refined from cleaned, partitioned and clustered tables (Silver layer) into highly aggregated business metrics (Gold layer). The final analytics are queried directly from the Gold tables by a **Streamlit** dashboard, ensuring users only interact with strictly validated data.

## 📂 Project Structure

This repository is strictly organized by pipeline stage, separating infrastructure, backend processing, and the frontend application into distinct, dependency-isolated environments:

```text
greece-internet-quality/
├── .github/workflows/      # GitHub Actions automation
│   ├── monthly_pipeline.yml            # Cron job for data ingestion & transformation
│   └── render-architecture-diagram.yml # Auto-compiles the D2 diagram on push
├── dashboard/              # Streamlit web application (Presentation Layer)
│   ├── .streamlit/         # Local secrets (auto-generated by Make)
│   ├── app.py              # Main Streamlit dashboard entry point
│   ├── data.py             # BigQuery connection and data retrieval logic
│   ├── figures.py          # Visualization and charting components
│   └── pyproject.toml      # uv dependencies for the dashboard
├── docs/                   # Documentation and assets
│   ├── architecture.d2     # Source code for the system architecture
│   └── architecture.svg    # Rendered SVG
├── infrastructure/         # Terraform Infrastructure as Code (IaC)
│   ├── compute.tf          # Cloud Run job definitions
│   ├── storage.tf          # GCS buckets and BigQuery datasets
│   ├── iam.tf              # Service accounts and IAM bindings
│   ├── github.tf           # GitHub Secrets management
│   ├── variables.tf        # Core configuration variables
│   └── secrets.auto.tfvars.example # Template for your GitHub PAT
├── ingest/                 # Python ingestion logic (Bronze Layer)
│   ├── src/                # API fetching scripts
│   ├── .env                # Local ingestor credentials (auto-generated)
│   └── pyproject.toml      # uv dependencies for ingestion
├── transform/              # PySpark data processing (Silver & Gold Layers)
│   ├── src/                # PySpark transformation scripts
│   ├── docker-compose.yml  # Local container testing environment
│   ├── Dockerfile          # Container definition for the serverless Cloud Run job
│   └── pyproject.toml      # uv dependencies for PySpark
├── Makefile                # Centralized command runner for local dev and deployment
└── README.md               # You are here!
```

## 🚀 Getting Started

### Prerequisites
Before you begin, ensure you have the following installed and configured:
* [Terraform](https://developer.hashicorp.com/terraform/install)
* [Make](https://www.gnu.org/software/make/)
* [uv](https://docs.astral.sh/uv/) (for Python environment management)
* [Google Cloud CLI](https://docs.cloud.google.com/sdk/docs/install-sdk)
* [Docker](https://docs.docker.com/engine/install/) (running on your local machine)
* A [GCP](https://console.cloud.google.com/) account with an active project.
* A Docker Hub account.

**GitHub Setup:**
1. Fork and/or download this repo.
2. Create a fine-grained GitHub Personal Access Token with `Secrets` permission (read and write) for your specific repo.
3. Copy the secrets template and add your token to the secrets file:
```bash
cd infrastructure 
cp secrets.auto.tfvars.example secrets.auto.tfvars
# Open secrets.auto.tfvars and paste your token inside
```

*(Note: All commands need to be executed in Bash. If you're on Windows, [Git Bash](https://git-scm.com/install/) is highly recommended).*

---

### Infrastructure
First, edit `infrastructure/variables.tf` with your desired project ID (the project must already exist in GCP), bucket name, dataset name, and GitHub repo details.

Once configured, log in to the Google Cloud CLI and provision the infrastructure:
```bash
make infra-up
```

> **⚠️ Expected Error:** You might get an error stating: *"Error waiting for Creating Job: Error code 5, message: Image 'mirror.gcr.io/<your_docker_image>' not found."* Don't worry! This happens because we haven't pushed the PySpark image to Docker Hub yet. The base job has still been created.

---

### Ingest

#### Local
Download the service account key required for data ingestion (which grants write permissions to the bucket) and generate a local `.env` file:
```bash
make create-ingestor-key
```

You can now run the Python ingestion script using `uv`. Here are a few examples:
```bash
# Ingest data for the current month
uv run --directory ingest python -m src.main monthly

# Ingest data for a specific month (e.g., Nov 2023)
uv run --directory ingest python -m src.main monthly 2023 11

# Backfill data between two dates
uv run --directory ingest python -m src.main backfill '2015-07' '2020-02'
```

#### Online
GitHub Actions is configured to automatically connect to GCP and ingest the previous month's data on the **1st day of every month at 04:14 UTC**.

---

### Transform

#### Local
Download the service account key for the transformation step (grants read access to the bucket and write access to BigQuery) and create the `.env` file for Docker:
```bash
make create-transformer-key
```

Build and run the PySpark transformation container locally:
```bash
make docker-run
```

#### Online
To run this in the cloud, you must push the image to Docker Hub. Build and push the image to docker hub: 
```bash
make image-push
```
This also addresses the error of `Image 'mirror.gcr.io/<your_docker_image>' not found` we had earlier.

*Note: It might take awhile for google cloud run to find the image because because Cloud Run resolves the container image to a specific SHA256 digest at deployment time. If the image didn't exist during your initial Terraform run, Cloud Run failed to deploy the revision, and will need some time before it resolves the image.*

---

### Dashboard

#### Local
Download the service account key for the dashboard (grants read/query access to BigQuery) and generate the `secrets.toml` file:
```bash
make create-dashboard-key
```

Launch the Streamlit app locally:
```bash
uv run --directory dashboard streamlit run app.py
```

#### Online
The production dashboard is hosted on Streamlit Cloud and can be viewed here:  
**[Greek Net Pulse Dashboard](https://greek-net-pulse.streamlit.app/)**
